## Why

Users need to explore ideas and run delivery concurrently without one stream clouding the other. Notifications should point into conversational context rather than becoming a detached task inbox.

## What Changes

- Add conversation lanes as coloured, addressable scopes and filter tags.
- Support single-lane, multi-lane, and global project views.
- Allow users and LLM tools to create lanes during conversation.
- Add addressable chat messages and action requests that notifications deep-link to.
- Preserve shared project memory while isolating UI attention by lane.

## Capabilities

### New Capabilities
- `conversation-lanes`: Lane-scoped chat, lane tags, lane creation tools, notifications, and inline action-request semantics.

### Modified Capabilities

## Impact

- Affects semantic event context, workbench projection, chat UI, notifications, role/tool routing, and future memory provenance.
