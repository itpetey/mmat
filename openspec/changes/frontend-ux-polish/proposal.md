## Why

The workbench needs to feel like a usable project workspace rather than a raw event demo. Users need clear next actions, robust feedback, accessible controls, and readable message/artefact rendering.

## What Changes

- Add next-action guidance and empty/error states.
- Add mention/lane/action affordances and keyboard support.
- Improve mobile, accessibility, loading, reconnect, and failure states.
- Add markdown/code rendering for chat and artefacts while preserving raw event inspection separately.

### New Capabilities
- `workbench-frontend-ux`: User-facing workbench interaction, accessibility, responsive layout, and rendering polish.
