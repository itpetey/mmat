## Why

The workbench needs clear operational behaviour: documented run commands, environment variables, bind errors, release packaging, and development shortcuts. Without that, it remains hard to start and diagnose.

## What Changes

- Document required and optional environment variables.
- Add clearer bind/configuration errors.
- Add dev/release run guidance and optional helper scripts.
- Decide and document prototype/MVP naming.
- Ensure release packaging includes static assets and Postgres requirements.
